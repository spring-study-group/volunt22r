Todo calendar 구현

요구사항

1. user 로그인 기능

2. 로그인 시 현재월과 todo list 나옴

3. 맨 위에 현재 월 양옆에는 화살표로 전달,다음달로 이동 가능

4. 날짜(일,day) 클릭시 새 todo 작성 추가

5. todo 수정 삭제 기능

6. todo 추가시 옵션으로 매일, 매주, 격주, 매월, 격월, 매년 알림 기능 추가

7. 알림 옵션으로는 n일/n시간/n분전 전 옵션 가능하도록)

8. 알림 기능은 로그인시 alert 알림(ex>해야할 일이 있습니다)

 

-UI는 자유롭게 구성(최대한 허접하게... Server side의 코드를 좀 더 아름답게 꾸밀 수 있도록 집중합시다)

-db 설계도 자유롭게

-dbms도 자유롭게(RDBMS도 좋고 NoSQL도 좋고... 다만 책 중반부 ORM쪽 하려면 RDBMS를 선택하는게 좋을 것 같음)

-기왕이면 Web Project로 Restful API까지 하는게 좋을 듯(Swing으로 구현해도 좋습니다만)
